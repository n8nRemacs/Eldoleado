# Стратегия работы с маркетологами — От пользователя до владельца вертикали

> **Концепция:** Маркетологи (SMM, SEO, контент) — идеальная аудитория для Автолида. Они становятся пользователями, а затем эволюционируют в владельцев собственных вертикалей.

---

## Два этапа эволюции маркетолога

### Этап 1: Маркетолог как пользователь
**Использует Автолид для себя:**
- Продаёт свои услуги (SMM, SEO, контент, таргет)
- Собирает лидов через Telegram-бот
- Прогревает email-сериями
- Конвертирует в клиентов

**Ценность:**
- Экономит время на рутине (квалификация, follow-up)
- Видит аналитику (CAC, LTV, конверсия)
- Не теряет лиды (AI отвечает мгновенно)

---

### Этап 2: Маркетолог как владелец вертикали

**Создаёт свою white-label вертикаль:**
- Например: "CRM для SMM-агентств"
- Подключает своих клиентов как тенантов
- Отслеживает их статистику (кампании, посты, метрики)
- Автоматизирует отчёты клиентам

**Монетизация (для платформы):**
- Маркетолог платит за каждого своего тенанта
- Небольшая сумма за тенанта (чтобы не убивать бизнес маркетолога)
- Или фиксированная плата за white label

**Ценность для маркетолога:**
- Единая система для всех клиентов
- Автоматические отчёты (еженедельно/ежемесячно)
- Профессиональный имидж ("агентство с платформой")
- Возможность допродавать доступ к CRM

---

## Почему маркетологи — идеальная аудитория

### 1. Понимают ценность автоматизации
- Сами занимаются маркетингом
- Знают боль потерянных лидов
- Ценят AI и технологии

### 2. Умеют продавать себя
- Активны в соцсетях
- Делают контент
- Уже привлекают клиентов

### 3. Имеют бюджет
- Средняя выручка: достаточно для подписки
- Понимают ROI
- Готовы платить за экономию времени

### 4. Становятся амбассадорами
- Рассказывают о продукте в соцсетях
- Привлекают других маркетологов
- Создают вирусный эффект

### 5. Путь к владельцам вертикалей
- Через несколько месяцев понимают систему
- Видят потенциал для своих клиентов
- Готовы создать свою вертикаль

## Ценность владения вертикалью (для маркетолога)

### Единая система для клиентов
- Все клиенты в одном dashboard
- Автоматические отчёты по всем кампаниям
- Централизованная аналитика

### Автоматизация рутины
- Email-отчёты клиентам (еженедельно/ежемесячно)
- Сбор метрик из соцсетей (Facebook, Instagram, VK)
- Напоминания о платежах

### Профессиональный имидж
- Свой бренд (white label)
- Клиентский портал для входа
- Не "маркетолог с Excel", а "агентство с платформой"

### Дополнительная выручка
- Может продавать доступ к CRM как допуслугу
- "Ведение соцсетей + доступ к аналитике = доп. плата"

### Месяц 4-6: Понимание системы

- Обращается с идеей создать white-label

- Платит тебе за каждого тенанта

## Монетизация (модели)

### Модель A: Роялти с тенанта
**Как работает:**
- Маркетолог платит небольшую сумму за каждого активного тенанта
- Выгодно при небольшом количестве клиентов

**Плюсы:**
- Низкий порог входа для маркетолога
- Ты зарабатываешь на масштабе (много маркетологов × их тенанты)

**Минусы:**
- Маркетолог с большой базой может уйти (если роялти станет дорого)

---

### Модель B: Фиксированная плата
**Как работает:**
- Маркетолог платит фикс за white label (безлимит тенантов)
- Выгодно при большой базе клиентов

**Плюсы:**
- Предсказуемый доход
- Маркетолог понимает затраты (не боится роста)

**Минусы:**
- Меньше дохода с маркетологов с большой базой

---

### Модель C: Гибридная (рекомендуемая)
**Как работает:**
- До определённого количества тенантов: роялти
- После порога: переход на фикс

**Пример:**
- До 30 тенантов: плата за каждого
- После 30 тенантов: фикс (безлимит)

**Плюсы:**
- Гибкость для маркетологов
- Ты не теряешь на крупных игроках
- Стимулирует рост (выгоднее расти дальше)

## Блок Звонки для маркетологов

### Зачем маркетологу запись звонков с AI?

**Этап 1: Для себя**
- Созвонился с клиентом
- AI извлекает: бюджет, боли, договорённости
- Автоматически создаёт задачи ("Отправить КП до пятницы")

**Этап 2: Для тенантов**
- Его клиенты (тенанты) тоже получают эту функцию
- Маркетолог видит качество звонков своих клиентов
- Может давать рекомендации (коучинг)

---

### Два варианта записи звонков

#### Вариант 1: Встроенная запись (Android app)
**Для кого:** Микробизнес, фрилансеры, маркетологи без телефонии

**Как работает:**
1. Маркетолог звонит через Android app
2. App автоматически записывает разговор
3. После завершения → отправка на сервер
4. AI извлекает сущности (бюджет, боли, договорённости)
5. Данные попадают в CRM, создаются задачи

**Преимущества:**
- Не нужна IP-телефония
- Работает с обычного смартфона
- Простой онбординг

---

#### Вариант 2: Интеграция с IP-телефонией
**Для кого:** Средний/крупный бизнес с существующей телефонией

**Как работает:**
1. Звонок через IP-телефонию (Asterisk, Zadarma, Mango)
2. Webhook после завершения звонка
3. Скачать запись из облака
4. AI извлекает сущности
5. Данные попадают в CRM

**Преимущества:**
- Не нужно переучивать сотрудников
- Записи хранятся у провайдера (не у тебя)
- Интеграция с привычной системой

---

### AI обработка звонков (техническая часть)

**Гибридный подход (экономичный):**

```
Запись звонка
  ↓
Vosk (локально, бесплатно) — базовая транскрибация
  ↓
Whisper (только сложные участки) — где Vosk не уверен
  ↓
Яндекс/Сбер (опционально) — для критичных моментов
  ↓
Claude Haiku (дешёвая модель) — извлечение сущностей
  ↓
Результат: структурированные данные
```

**Почему это дёшево:**
- Не транскрибируем весь разговор через платные API
- Только извлекаем сущности (малый объём токенов)
- Vosk делает основную работу локально (бесплатно)

---

### Что извлекаем из звонков

**Контекст для CRM:**
- Имя клиента
- Компания
- Бюджет (если упомянут)
- Срочность ("нужно вчера", "думаем до конца месяца")
- Боли ("не хватает лидов", "дорогой трафик")
- Возражения ("дорого", "уже работаем с другим")
- Договорённости ("созвонимся в пятницу", "пришлю КП")

**Автоматические действия:**
- Создать задачу: "Отправить КП до пятницы"
- Обновить статус: "Квалификация → Переговоры"
- Напомнить: "Позвонить 15.12 в 14:00"
- Добавить тег: "Возражение: дорого"

---

### Хранение записей (минималистично)

**Что НЕ храним долго:**
- ❌ Аудиофайлы звонков (удаляем через 7-30 дней)
- ❌ Полные транскрипты (удаляем через 30 дней)

**Что храним постоянно:**
- ✅ Извлечённые сущности (JSON)
- ✅ Metadata: дата, длительность, от кого, кому
- ✅ Ключевые фразы (боли, возражения, договорённости)

**Экономия на хранении:**
- Аудиофайл 10 минут = ~5 MB
- Извлечённые данные = ~2 KB
- Экономия в 2500 раз на одном звонке

### 4. Путь к владельцам вертикалей

- Создают экосистему
- Приводят своих тенантов

### 5. Proof of platform
- Для инвесторов: не просто продукт, а платформа
- Разные ниши (SMM, SEO, контент, таргет)
- Сетевой эффект
