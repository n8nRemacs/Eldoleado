## 0. CORE POSITIONING

### Главный инсайт

**Удержание клиента в 60-600 раз дешевле привлечения нового.**

| | Привлечение | Удержание |
|---|-------------|-----------|
| Стоимость | 300-3000₽ за клиента | 5₽ в год |
| Разница | — | **в 60-600 раз дешевле** |
| Кто этим занимается | Маркетологи, агентства, реклама | Почти никто системно |
| Куда уходят бюджеты | 90% | 10% или ноль |

### Парадокс рынка

Бизнес тратит миллионы на привлечение и ноль на удержание. Клиенты утекают, бизнес снова тратит миллионы на привлечение.

**Метафора:** Это как носить воду в решете и покупать всё больше воды вместо того, чтобы залатать дыры.

### Что такое Eldoleado

**Eldoleado = заплатка для решета.**

Не «ещё одна CRM». Не «автоматизация продаж». А **система, которая не даёт клиентам утекать**.

### Для кого

Любой бизнес с повторными покупками:

| Сегмент | Примеры |
|---------|---------|
| Услуги по необходимости | Ремонт техники, сантехник, авторемонт, ветеринария |
| Услуги периодические | Салоны, клининг, фитнес |
| Розница с выбором | Одежда, электроника, мебель |
| Розница периодическая | Доставка еды, e-commerce |
| Локальный бизнес | Кафе, магазины у дома |

### Конкурентное преимущество

Все CRM-системы заточены на:
- Управление воронкой продаж (привлечение)
- Автоматизацию процессов (операции)
- Аналитику (отчёты)

Никто не фокусируется на **удержании существующих клиентов как главной задаче**.

Eldoleado — первая система, где удержание = core function, а не побочная фича.

## 1. СЛЕПОК БРАУЗЕРА (Browser Fingerprint)

### Что это
Уникальный идентификатор устройства/браузера, формируемый из комбинации технических характеристик. В отличие от cookies, не удаляется пользователем и сохраняется даже в режиме инкогнито.

### Какие данные можно получить

| Категория | Параметры | Описание |
|-----------|-----------|----------|
| **Устройство** | Screen resolution, color depth, pixel ratio | Разрешение экрана, глубина цвета |
| **Устройство** | Hardware concurrency (CPU cores) | Количество ядер процессора |
| **Устройство** | Device memory | Объём оперативной памяти |
| **Браузер** | User-Agent string | Браузер, версия, ОС |
| **Браузер** | Language, timezone, timezone offset | Язык, часовой пояс |
| **Браузер** | Installed plugins, MIME types | Установленные плагины |
| **Браузер** | Do Not Track setting | Настройки приватности |
| **Браузер** | Cookies/localStorage enabled | Поддержка хранилищ |
| **Графика** | Canvas fingerprint | Уникальный рендер 2D-графики |
| **Графика** | WebGL fingerprint, GPU vendor/renderer | Информация о видеокарте |
| **Аудио** | AudioContext fingerprint | Уникальная обработка аудио |
| **Шрифты** | Installed fonts list | Список установленных шрифтов |
| **Сеть** | IP address (серверная часть) | IP-адрес, геолокация |

### Условия получения

| Условие | Что получаем |
|---------|--------------|
| Переход по любой ссылке на ваш сайт | Полный fingerprint автоматически |
| JavaScript отключён | Только IP и User-Agent (ограниченно) |
| Tor/VPN/Antidetect браузеры | Может быть искажён или неуникален |

### Точность идентификации
- 83-94% браузеров имеют уникальный fingerprint
- Одинаковые устройства (например, два iPhone 15 с Safari) могут иметь одинаковый fingerprint
- Fingerprint может измениться при обновлении ОС/браузера/драйверов

### Рекомендуемые инструменты
- **FingerprintJS** (open-source) — бесплатно, базовый функционал
- **Fingerprint Pro** (платный) — выше точность, серверная обработка

### Как использовать для склейки
Клиент переходит по персональной ссылке из WhatsApp/Telegram → fingerprint связывается с номером телефона → теперь можно показывать ретаргетинг даже если клиент не авторизован.

## 2. VK (ВКонтакте)

### Способы получения данных

#### 2.1 VK Login (OAuth авторизация)

При авторизации через VK на вашем сайте можно запросить различные права (scopes):

| Scope | Что даёт | Требует подтверждения |
|-------|----------|----------------------|
| `basic` | VK ID, имя, фамилия, фото | Нет |
| `email` | Email пользователя | Да |
| `phone` | Номер телефона | Да (только standalone apps) |
| `friends` | Список друзей | Да |
| `photos` | Доступ к фотографиям | Да |
| `wall` | Доступ к стене | Да |
| `groups` | Список групп | Да |
| `offline` | Токен не истекает | Да |

**Данные из users.get API:**

| Поле | Описание | Условие |
|------|----------|---------|
| `id` | Уникальный VK ID | Всегда |
| `first_name`, `last_name` | Имя, фамилия | Всегда |
| `photo_*` | Фотографии профиля (50-400px) | Всегда |
| `sex` | Пол (1-жен, 2-муж, 0-не указан) | Если открыто |
| `bdate` | Дата рождения (DD.MM или DD.MM.YYYY) | Если открыто |
| `city`, `country` | Город, страна | Если открыто |
| `contacts` | Телефон, Skype | Если открыто (редко) |
| `education` | Университет, факультет, год выпуска | Если открыто |
| `universities`, `schools` | История образования | Если открыто |
| `online` | Сейчас онлайн (1/0) | Всегда |
| `last_seen` | Последний визит (timestamp + platform) | Если открыто |
| `verified` | Верифицированный аккаунт | Всегда |
| `is_premium` | Подписка VK Premium | Всегда |

#### 2.2 VK Pixel (ретаргетинг)

Пиксель VK устанавливается на сайт и собирает:

| Данные | Описание |
|--------|----------|
| Посещённые страницы | URL, время на странице |
| Действия | Клики, добавление в корзину, покупки |
| VK ID | Если пользователь авторизован в VK |
| Поисковые запросы | При настройке события |

**Возможности пикселя:**
- Создание аудиторий для ретаргетинга
- Динамический ретаргетинг (показ просмотренных товаров)
- Look-alike аудитории
- Оптимизация рекламы под конверсии

#### 2.3 VK Группы/Сообщества

Если клиент подписался на вашу группу:

| Данные | Условие |
|--------|---------|
| VK ID | Всегда |
| Публичный профиль | Всегда |
| Участие в обсуждениях | При активности |

## 3. WHATSAPP

### 3.1 WhatsApp Business App (бесплатно)

| Данные | Как получить |
|--------|--------------|
| Номер телефона | Автоматически при входящем сообщении |
| Имя контакта | Из профиля WhatsApp (если указано) |
| Фото профиля | Видно в чате |
| Статус "был онлайн" | Видно в чате |

### 3.2 WhatsApp Business API

| Данные | Условие |
|--------|---------|
| Номер телефона (E.164 формат) | При любом сообщении |
| Имя профиля | При любом сообщении |
| Фото профиля | Не доступно через API |
| Медиа-файлы | Если клиент отправил |

**Ограничения API:**
- Нельзя получить данные без инициации диалога клиентом
- Первое сообщение от бизнеса — только по шаблону
- 24-часовое окно для свободных сообщений после ответа клиента
- Требуется opt-in от клиента

### Стратегия обогащения через WhatsApp

1. Клиент написал → получили номер телефона
2. Отправляем ссылку на полезный контент → получаем fingerprint
3. Просим подписаться на VK/Telegram → получаем дополнительный канал

## 4. TELEGRAM

### 4.1 Telegram Bot API

При взаимодействии с ботом доступны данные из объекта `User`:

| Поле | Описание | Условие |
|------|----------|---------|
| `id` | Уникальный Telegram ID | Всегда |
| `first_name` | Имя | Всегда |
| `last_name` | Фамилия | Опционально (если указано) |
| `username` | @username | Опционально (если указано) |
| `language_code` | Языковой код (IETF) | Опционально |
| `is_premium` | Telegram Premium | Доступно |
| `photo` | Фото профиля | Через отдельный запрос |

### 4.2 Получение номера телефона

Telegram позволяет запросить номер телефона через специальную кнопку "Поделиться контактом". Клиент должен явно согласиться — без согласия номер не передаётся.

| Условие | Результат |
|---------|-----------|
| Клиент нажал кнопку и согласился | Получаем номер телефона |
| Клиент отказался | Номер не получен |

**Важно:** Telegram НЕ позволяет получить номер телефона по username или ID без явного согласия пользователя.

### 4.3 Telegram Login Widget (на сайте)

При авторизации через Telegram на сайте:

| Данные | Условие |
|--------|---------|
| Telegram ID | Всегда |
| Имя, фамилия | Всегда |
| Username | Если есть |
| Фото | Если есть |
| Номер телефона | НЕ передаётся |

## 5. SMS

### Какие данные можно получить

| Данные | Источник |
|--------|----------|
| Номер телефона | Исходные данные для отправки |
| Доставлено/прочитано | Статус от оператора (не всегда) |
| Оператор связи | Carrier Lookup API |
| Тип номера | Мобильный/городской/VoIP |
| Регион | По коду оператора |

### Carrier Lookup

Сервисы типа TextMagic, Twilio позволяют определить:
- Название оператора (МТС, Билайн, Мегафон...)
- Тип линии (mobile, landline, VoIP)
- Страну и регион
- Портирован ли номер

### Ограничения SMS
- Нельзя получить имя владельца по номеру
- Требуется opt-in для маркетинговых сообщений
- Высокая стоимость относительно мессенджеров

## 6. ТЕЛЕФОННЫЙ ЗВОНОК

### 6.1 Caller ID / CNAM

При входящем звонке можно получить:

| Данные | Источник |
|--------|----------|
| Номер звонящего (ANI) | Автоматически |
| Имя звонящего (CNAM) | Через CNAM Lookup (до 15 символов) |

**CNAM (Caller Name):**
- Работает только в США и Канаде
- Имя берётся из базы данных оператора
- Может быть неактуальным (операторы обновляют редко)
- Не все операторы передают CNAM

### 6.2 Данные из разговора

| Данные | Как получить |
|--------|--------------|
| День рождения | Спросить (для скидки/поздравления) |
| Предпочтения | Записать в заметки |
| Email | Спросить для отправки информации |
| Адрес | При необходимости доставки |

### 6.3 Интеграция с CRM

При интеграции телефонии с CRM:
- Автоматическое создание карточки клиента
- Запись разговоров
- История звонков
- Длительность, время звонка

## 11. РЕТАРГЕТИНГ (РСЯ / VK ADS)

### Концепция

Ретаргетинг через рекламные сети — самый дешёвый способ оставаться в поле зрения клиента. Не продажа, а напоминание: «мы существуем, мы рядом».

### Экономика

| Параметр | Значение |
|----------|----------|
| Средний CPM (РСЯ) | 30-150₽ |
| Стоимость 1 показа | 0,03-0,15₽ |
| Показов в год (раз в 10 дней) | 36 |
| **Стоимость прогрева 1 клиента в год** | **3-6₽** |

Для сравнения: стоимость привлечения нового клиента в сервисном бизнесе — 300-3000₽. Удержание через ретаргетинг в 60-600 раз дешевле. И несколько лет работы маркетолога.

### Экономика возврата «потерянных» лидов

Это ключевой момент. При конверсии 10-15% бизнес теряет 85% лидов. Они уже пришли, уже был контакт — но не купили. За каждого из них заплачено 300-3000₽.

Классический подход: забыть и привлекать новых за те же деньги.

Подход Eldoleado: догревать потерянных через ретаргетинг за 5₽/год.

**Пример:**

| Показатель | Значение |
|------------|----------|
| Лидов в год | 1000 |
| Конверсия | 15% |
| Клиентов | 150 |
| Потерянных лидов | 850 |
| Стоимость привлечения лида | 500₽ |
| Потрачено на потерянных | 425 000₽ |
| | |
| Догрев 850 лидов год (ретаргетинг) | 4 250₽ |
| Дополнительная конверсия 5% | 42 клиента |
| Стоимость этих клиентов | ~100₽/клиент |

Вместо 500₽ за нового клиента — 100₽ за «реанимированного». И это клиенты, за которых уже заплачено.

**Вывод:** Ретаргетинг — это не просто удержание. Это способ выжать максимум из уже оплаченного трафика.

### Как это работает в Eldoleado

1. Мастер загружает базу клиентов (телефоны или email)
2. Система создаёт аудиторию в РСЯ / VK Ads
3. Клиентам показывается реклама с заданной частотой (например, раз в 10 дней)
4. Креативы: не «купи сейчас», а напоминание о бренде, полезные советы, сезонные предложения

### Преимущества канала

- **Пассивность** — работает само, не требует действий от мастера
- **Дешевизна** — 3-6₽ в год на клиента
- **Масштаб** — можно охватить всю базу одновременно
- **Ненавязчивость** — клиент видит рекламу в своём инфополе, не получает личных сообщений

### Ограничения

- Нужна база с телефонами или email (минимум 100-1000 контактов для запуска)
- Не все клиенты матчатся с аудиториями рекламных сетей
- Нет обратной связи — не знаем, кто именно видел рекламу

### Место в графе обогащения

Ретаргетинг — это «фоновый» канал касания. Он не заменяет личные сообщения, а дополняет их. Клиент получает WhatsApp раз в месяц + видит рекламу раз в 10 дней = постоянное присутствие в инфополе.

## 11.1. РАСШИРЕНИЕ РЫНКА: УСЛУГИ ПО НЕОБХОДИМОСТИ

### Два типа сервисного бизнеса

| Периодические услуги | Услуги по необходимости |
|---------------------|------------------------|
| Парикмахерская | Ремонт телефонов |
| Маникюр | Ремонт бытовой техники |
| Массаж | Сантехник / электрик |
| Фитнес | Ремонт авто |
| Косметолог | Ремонт одежды / обуви |
| Стоматология (профилактика) | Стоматология (острая боль) |
| | Ветеринария (лечение) |
| | Клининг (разовый) |

### Почему услугам по необходимости сложнее удерживать клиентов

1. **Нет регулярности** — клиент приходит раз в год или реже
2. **Забывают** — между визитами проходит слишком много времени
3. **Конкуренция в момент поломки** — клиент гуглит заново каждый раз
4. **Нет естественного повода напомнить** — «приходи подстричься» работает, «приходи сломай телефон» — нет

### Почему Eldoleado критичен именно для этого сегмента

Парикмахерская может просто ждать — клиент вернётся через месяц по привычке.

Ремонт телефонов ждать не может — клиент вернётся через год и забудет, к кому ходил. В момент поломки он откроет карты/поиск и выберет того, кто окажется сверху или кого вспомнит.

**Задача Eldoleado:** сделать так, чтобы в момент необходимости клиент вспомнил именно этого мастера.

Инструменты:
- Ретаргетинг (фоновое присутствие в инфополе)
- Сезонные напоминания («проверь телефон перед отпуском»)
- Поздравления и касания (день рождения, праздники)
- Полезный контент (советы по уходу за техникой)

### Расширенный TAM (Total Addressable Market)

Eldoleado — не «CRM для ремонта телефонов», а **система удержания клиентов для бизнеса по необходимости**:

| Сегмент | Примеры | Боль |
|---------|---------|------|
| Ремонт техники | Телефоны, компьютеры, бытовая техника | Клиент забывает между поломками |
| Бытовые услуги | Сантехник, электрик, мастер на час | Вызывают раз в год, ищут заново |
| Авторемонт | СТО, шиномонтаж, детейлинг | Сезонность + длинные паузы |
| Ремонт вещей | Одежда, обувь, сумки | Редкие обращения |
| Ветеринария | Лечение (не профилактика) | Приходят только при болезни |
| Клининг | Разовый, генеральная уборка | Нет регулярного графика |

### Стратегическое преимущество

Конкуренты (AmoCRM, Битрикс, RetailCRM) заточены под:
- E-commerce (регулярные покупки)
- B2B-продажи (длинный цикл, но активная работа)
- Периодические услуги (салоны, фитнес)

Никто не решает задачу **«как не потерять клиента, который придёт через год»**.

Eldoleado закрывает эту нишу.

### Ключевой selling point

**Сравнение затрат на удержание 5000 клиентов:**

| Вариант | Стоимость в год |
|---------|-----------------|
| Маркетолог на аутсорсе | 180 000 — 360 000₽ |
| Штатный маркетолог | 600 000 — 1 000 000₽ |
| **Ретаргетинг через Eldoleado** | **25 000₽** |

При этом маркетолог тратит бюджет на привлечение новых (300-3000₽ за клиента), а система удерживает уже существующих за 5₽.

**Питч для мастерской:**

«У тебя уже есть 5000 клиентов. Ты заплатил за каждого из них. Сейчас они тебя забывают и уходят к конкурентам. За 2000₽ в месяц они будут видеть тебя постоянно и возвращаться. Это дешевле, чем один пост у блогера.»

Телефонный разговор → день рождения, предпочтения, история. Email-подписка → email для рассылок. Визит на точку → история покупок, средний чек.

## 15. ГРАФ ОБОГАЩЕНИЯ ПРОФИЛЯ

### Концепция

Точки касания с клиентом образуют граф:
- **Вершины** — каналы/идентификаторы (WhatsApp, Telegram, VK, Fingerprint, Email и т.д.)
- **Рёбра** — механизмы перехода из одного канала в другой

Задача системы — провести клиента по графу, собирая максимум точек касания и объединяя их в единый профиль.

### Карта вершин (точек касания)

| Вершина | Что даёт | Что нужно для входа |
|---------|----------|---------------------|
| **Авито чат** | Только чат Авито | Объявление |
| **2ГИС** | Телефон (холодный) | Парсинг базы |
| **Телефон** | Номер, CNAM (США/Канада) | Любой источник номера |
| **WhatsApp** | Телефон, имя, фото профиля | Номер телефона |
| **Telegram Bot** | TG ID, username, имя, язык | Номер или username |
| **SMS** | Статус доставки, оператор | Номер телефона |
| **Fingerprint** | Устройство, браузер, гео, GPU | Клик по ссылке на свой сайт |
| **VK ID** | ID, имя, ДР, город, фото, образование | OAuth или пиксель |
| **VK Pixel** | Поведение на сайте, ретаргетинг | Установка пикселя |
| **Email** | Email, открытия, клики | Форма, запрос в разговоре |
| **Визит на точку** | История покупок, средний чек, предпочтения | Клиент пришёл физически |
| **Социальная связь** | Связь с другими клиентами | Реферальная программа |
| **Ретаргетинг (РСЯ/VK)** | Показы рекламы, напоминание о бренде | Загрузка базы в рекламный кабинет |

### Карта рёбер (механизмы перехода)

| Из | В | Механизм | Примерная конверсия |
|----|---|----------|---------------------|
| Авито чат | WhatsApp | "Напишите в WhatsApp для быстрого ответа: +7..." | 30-40% |
| Авито чат | Звонок | Клиент сам звонит по номеру из объявления | 20-30% |
| WhatsApp | Fingerprint | Ссылка на полезный контент/сайт | 40-60% |
| WhatsApp | Telegram | "Подпишитесь на канал, там эксклюзивные акции" | 10-20% |
| WhatsApp | VK | "Наши работы в группе VK" | 10-15% |
| Telegram | Телефон | Кнопка "Поделиться контактом" (с согласия) | 20-30% |
| Fingerprint | VK ID | VK Pixel на сайте (если авторизован в VK) | автоматически |
| Fingerprint | Email | Форма подписки, поп-ап на сайте | 5-15% |
| Звонок | Любое | Спросить в разговоре (email, ДР, предпочтения) | зависит от контекста |
| Визит | Любое | Анкета, программа лояльности, разговор | 50-70% |
| Любой | Социальная связь | Реферальная программа | см. раздел ниже |

### Визуализация графа

Путь клиента по графу обогащения:

Авито (чат) → WhatsApp / Telegram / Звонок → Телефон (подтверждён) → Fingerprint (ссылка на сайт) → VK ID / Email / TG ID

Каждый переход открывает новые точки касания и данные о клиенте.

### Принципы работы с графом

1. **Не гнаться за 100% конверсией** — клиент не кликнул по ссылке сегодня, кликнет в следующем касании. Главное — отношения, а не сбор данных.

2. **Естественность** — переходы по рёбрам должны быть полезны клиенту ("посмотрите наши работы", "там акции"), а не выглядеть как сбор информации.

3. **Постепенность** — не пытаться получить всё за одно касание. Каждое взаимодействие может добавить один канал.

4. **Приоритет** — сначала надёжные каналы связи (телефон, мессенджеры), потом дополнительные (fingerprint, VK для ретаргетинга).

## 16. СОЦИАЛЬНЫЕ СВЯЗИ (ЧЕЛОВЕК → ЧЕЛОВЕК)

### Концепция

Социальные связи между клиентами — отдельный слой графа. Это не канал касания напрямую, но:
- Источник новых клиентов (сарафан)
- Способ реактивации через близких
- Данные для аналитики "заразности" клиентов

### Типы связей

| Тип связи | Как образуется | Как используем |
|-----------|---------------|----------------|
| Реферал | Клиент привёл друга по коду/ссылке | Бонусы, статистика сарафана |
| Семья | Одинаковый адрес, фамилия, сам сказал | Комплексные предложения |
| Коллеги | Корпоративный заказ, один адрес офиса | Выход на B2B |

### Реферальная программа

**Главный механизм получения связей — реферальная программа.** Без неловких вопросов "откуда узнали", всё происходит автоматически.

**Механика:**

1. Клиент получает персональный код или ссылку (например, "IVAN10" или "remont.ru/r/ivan")
2. Друг приходит, называет код → получает скидку
3. Система автоматически даёт скидку новому клиенту, связывает карточки клиентов, и (опционально) начисляет бонус пригласившему

**Варианты мотивации:**

| Схема | Новый клиент | Пригласивший | Сложность учёта |
|-------|--------------|--------------|-----------------|
| Односторонняя | Скидка 10% | Ничего | Минимальная |
| Двусторонняя | Скидка 10% | Скидка 10% на след. визит | Средняя |
| Накопительная | Скидка 10% | Баллы за каждого | Высокая |

**Рекомендация для микробизнеса:** начинать с односторонней схемы. Связь фиксируется, клиент получает выгоду, учёт минимальный.

### Данные связи

Для каждой связи система хранит:
- Кто кого привёл (направление связи)
- Тип связи (реферал, семья, коллеги)
- Реферальный код, если использовался
- Дата образования связи
- Источник информации о связи

### Использование связей

**Автоматическое (система делает сама):**
- Связывание карточек при использовании реф. кода
- Подсчёт статистики: сколько привёл каждый клиент
- Определение "заразных" клиентов (приводят много)
- Аналитика: работает ли сарафан в этой вертикали

**Полуавтоматическое (система подсказывает мастеру):**

Пример: "Иван Иванович не отвечает 30 дней. Завтра визит Иван Петровича (друг, который его привёл). Напомнить спросить про Иван Ивановича?"

Мастер сам решает — уместно спросить или нет. Система только напоминает о связи.

**Чего НЕ делаем:**
- Не просим клиента "передай другу, что пора к нам"
- Не используем связи для навязчивых действий
- Не раскрываем клиентам информацию друг о друге

### Связь в графе обогащения

Когда новый клиент приходит с реферальным кодом (например, IVAN10), система автоматически создаёт вершину "Социальная связь" и связывает его с Иван Петровичем как реферала.

### Метрики сарафана

| Метрика | Что показывает |
|---------|----------------|
| Количество рефералов | Сколько клиентов пришло по рекомендациям |
| Конверсия рефералов | % рефералов, ставших постоянными |
| Топ "амбассадоров" | Клиенты, которые привели больше всех |
| LTV реферала vs обычного | Ценнее ли клиенты по рекомендации |
| Виральность вертикали | Работает ли сарафан в этой нише |

## 17. АРХИТЕКТУРА СИСТЕМЫ ОБОГАЩЕНИЯ

### Три слоя

**СЛОЙ 1: АГРЕГАЦИЯ**
Все точки касания → единый профиль клиента
- Склейка по телефону (WhatsApp + TG + SMS = 1 клиент)
- Склейка по fingerprint (браузер → телефон)
- Склейка по реф. коду (связь между клиентами)
- AI-склейка (похожие данные → предложение объединить)

**СЛОЙ 2: ГРАФ ОБОГАЩЕНИЯ**
Вершины (каналы) + Рёбра (механизмы перехода)
- Карта: какие каналы есть, каких не хватает
- Рекомендации: "отправьте ссылку → получите fingerprint"
- A/B тестирование рёбер (какие механизмы работают)

**СЛОЙ 3: ЦЕПОЧКИ КАСАНИЙ**
Сценарии взаимодействия с встроенным обогащением
- Каждое N-е касание содержит попытку получить канал
- Автоматический выбор канала для касания
- Fallback: если не ответил в WhatsApp → попробуй Telegram

## Заключение

### Ключевые принципы Eldoleado

1. **Цель — удержание клиента и отношения**, а не сбор данных. Каналы и fingerprint — инструменты доставки касаний, не самоцель.

2. **Активное управление обогащением профиля.** Каждое касание может (но не обязано) открывать новый канал связи.

3. **Граф обогащения** — вершины (каналы) и рёбра (механизмы перехода). Система ведёт клиента по графу естественным путём.

4. **Социальные связи** через реферальную программу. Без неловких вопросов, автоматическая фиксация связей.

5. **Постепенность и естественность.** Не гнаться за 100% конверсией, не собирать всё за одно касание. Отношения важнее данных.

Чем больше каналов связано с клиентом — тем сложнее его потерять и тем персональнее могут быть касания. Но даже один надёжный канал (WhatsApp с ответами) ценнее пяти "мёртвых".
