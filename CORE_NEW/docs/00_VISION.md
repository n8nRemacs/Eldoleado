# ELDOLEADO: Видение продукта

## Проблема

**Традиционные CRM убивают живое общение.**

Микро и малый бизнес выбирает: либо живое общение с клиентами (и хаос в голове), либо CRM (и бесконечное заполнение полей, канбаны, статусы).

**Результат:**
- CRM покупают, но не используют
- Данные не заполняют или заполняют мусором
- Менеджеры ненавидят систему
- Владелец не получает отчётность

---

## Решение

**Люди общаются. Машина ведёт учёт.**

Так работают люди на самом деле: общаются с клиентами, записывают что-то в произвольной форме или запоминают, снова общаются, продают, покупают. Они не ведут таблицы и не заполняют поля — они просто работают.

**Традиционные CRM ломают это.** Они стандартизируют живое общение, заставляют человека работать на систему.

**Eldoleado переворачивает это:**
- Человеку — общение (как он привык)
- Машине — рутина, учёт, таблицы, поля

Снаружи — обычный омниканальный мессенджер: telegram, whatsapp, звонки, почта, SMS.
Внутри — AI, который:

- **Помнит всё** — всю историю с каждым клиентом
- **Понимает контекст** — о чём речь, что нужно клиенту
- **Подсказывает** — что ответить, что уточнить, какую цену назвать
- **Берёт на себя рутину** — сам заполняет, сам двигает, сам напоминает
- **Даёт отчётность** — без единого заполненного поля вручную

## Ключевые принципы

### 1. Общение первично

```
НЕ ТАК:                          А ТАК:
┌─────────────────┐              ┌─────────────────┐
│ Создать заявку  │              │ Привет! У меня  │
│ Заполнить поля  │              │ iPhone разбился │
│ Выбрать статус  │              └────────┬────────┘
│ Прикрепить клиента              │
│ ...             │                       │
└─────────────────┘              ┌────────▼────────┐
                                 │   AI понял:     │
                                 │   - клиент      │
                                 │   - устройство  │
                                 │   - проблема    │
                                 │   - намерение   │
                                 └─────────────────┘
```

Пользователь просто общается. Система сама:
- Определяет клиента
- Понимает о чём речь
- Создаёт/обновляет записи
- Двигает по воронке

### 2. Без канбанов и форм

**Канбан — это визуализация процесса для человека.**

Если AI ведёт процесс сам, канбан не нужен. Система знает:
- На каком этапе каждый клиент
- Что нужно сделать следующим
- Когда напомнить

> **Если кому-то нужен канбан** — подключим по API (Trello, Notion, Planfix). Или сделаем свой когда-нибудь. Но не на старте. Или вообще никогда.

**Формы — это способ ввода данных.**

Если AI извлекает данные из разговора, формы не нужны. Максимум — подтверждение: "Правильно понял: iPhone 14 Pro, разбит экран?"

### 3. Отчётность из воздуха

Раз AI понимает каждое сообщение, он знает:
- Сколько обращений по ремонту / покупке / вопросам
- Средний чек
- Конверсия по этапам
- Время ответа
- Повторные клиенты

**Без единого заполненного поля вручную.**

### 4. AI помогает, не заменяет

```
┌─────────────────────────────────────────────────────┐
│ Клиент: сколько стоит поменять экран на 14 про?     │
├─────────────────────────────────────────────────────┤
│ 💡 Подсказка:                                       │
│ Оригинал: 25000₽ (в наличии)                        │
│ Копия AAA: 12000₽ (в наличии)                       │
│ Копия AA: 8000₽ (под заказ 2 дня)                   │
│                                                     │
│ [Отправить прайс] [Уточнить модель] [Своё]          │
└─────────────────────────────────────────────────────┘
```

AI подсказывает, человек решает. Или AI отвечает сам (если настроено).

## MVP (первая версия)

### Что делаем:
1. Омниканальный мессенджер (telegram, whatsapp для начала)
2. AI извлекает: клиент, устройство, проблема, намерение
3. Простой граф: Client → Device → Problem
4. Автоматические статусы по ключевым словам
5. Подсказки менеджеру (цены, наличие, слоты)
6. Базовая отчётность

### Чего НЕ делаем на MVP:
- Сложные сценарии и прерывания
- Полную автоматизацию ответов
- Интеграции с 1С, складом
- Голосовые каналы

---

## Целевая аудитория

**Микро и малый бизнес сферы услуг:**
- Ремонт техники (телефоны, ноутбуки, бытовая)
- Автосервисы
- Салоны красоты
- Стоматологии
- Ветеринарные клиники
- Фотостудии

**Общее:**
- Много коммуникации с клиентами
- Повторные обращения
- Нужна история и контекст
- Ненавидят заполнять CRM

---

## Почему это сработает

1. **Барьер входа = 0** — просто общайся как раньше
2. **Ценность сразу** — история, поиск, контекст
3. **Растущая ценность** — чем больше общения, тем умнее система
4. **Нет сопротивления** — не надо менять привычки
5. **Отчётность бесплатно** — владелец счастлив
